# Panel 流式 Asset 聚合方案

## 问题

Panel 消息流中，流式输出（TTS 音频、监控视频）会产生大量独立 block，淹没消息列表。

## 设计

流式 block 不逐条显示，只在消息流中插入两条标记：

```
[▶ 流开始]  ← 点击打开实时预览弹窗
  (中间的 blocks 静默入库，不在消息流中显示)
[■ 流结束]  ← 点击查看聚合后的完整 asset
```

### 信号来源

- **方案 A（推荐）**: 节点在 metadata 中标注 `{ "stream": "start" }` / `{ "stream": "end" }`
- **方案 B（fallback）**: Panel Store 根据规则推断（同一 output_id 连续 N 个 block → 自动识别为流）
- A 和 B 共存，A 优先

### 关键行为

| 阶段 | 消息流 | 后台 |
|---|---|---|
| 流开始 | 显示「▶ 启动」+ 实时预览按钮 | 标记 stream_id，后续 block 静默入库 |
| 流进行中 | 不产生新消息 | 持续接收 block，推送给已打开的预览弹窗 |
| 流结束 | 显示「■ 结束」+ 查看按钮 | 聚合所有 block 为完整 asset |

### 适用场景

- **有限聚合**（TTS）: 多个 block → 一个完整音频
- **无限流**（监控）: 持续 block → 实时预览 + 结束后聚合录像
