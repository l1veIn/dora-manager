# Records

This folder stores ongoing operational records for long-running project work.

Use it for materials that are append-only or cycle-based, such as:
- dogfooding rounds
- steering meetings
- acceptance logs
- recovery notes

Current records:
- `ux-test-rounds.md`
- `steering-cycles.md`

Rule:
- keep durable product docs, designs, and specifications in `docs/`
- keep iterative execution logs and round records in `docs/records/`
